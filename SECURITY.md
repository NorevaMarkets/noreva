# Security Policy

## Reporting a Vulnerability

The Noreva team takes security seriously. If you discover a security vulnerability, please report it responsibly.

### How to Report

**DO NOT** create a public GitHub issue for security vulnerabilities.

Instead, please report security issues via:
- Or via direct message on [X](https://x.com)

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution Target**: Within 30 days (depending on severity)

## Scope

This security policy applies to:
- The Noreva web application (noreva.markets)
- The Noreva documentation (docs.noreva.markets)
- This GitHub repository

## Out of Scope

- Third-party services (Jupiter, Solana, wallet providers)
- Social engineering attacks
- Denial of service attacks

## Safe Harbor

We consider security research conducted in accordance with this policy to be:
- Authorized
- Not a violation of any applicable laws
- Not a violation of our Terms of Service

We will not pursue legal action against researchers who follow this policy.

## Security Best Practices for Users

1. **Wallet Security**: Never share your seed phrase or private keys
2. **Verify URLs**: Always verify you're on noreva.markets
3. **Transaction Review**: Review all transaction details before signing
4. **Official Links**: Only use links from our official channels
